6. QR Onboarding Flow

Use QR setup when you want a phone to receive the server URL and login details without typing them manually.

  1. Log in to the web admin as admin@slid.local.
  2. Go to /admin/users.
  3. Open the setup QR page for officer@slid.local.
  4. Enter the server URL the phone can reach.
  5. Generate the setup QR.
  6. On Android, tap Scan Setup QR.
  7. Scan the QR code.
  8. Confirm the server URL, email, and device name are filled.
  9. Sign in.
Generating a setup QR can reset the officer password to a temporary password inside the QR. Use the newest QR, or reseed locally to return to Officer123!.

7. End-To-End Field Test

  1. Sign in to Android as officer@slid.local.
  2. Open a downloaded form.
  3. Confirm the form is split into step screens.
  4. Fill the required fields.
  5. Use date and time pickers where available.
  6. Use MRZ scan fields when testing travel-document reporting.
  7. Save a draft and reopen it.
  8. Finalize and send while online.
  9. Turn off network and finalize another report.
  10. Confirm the app clearly says the report is saved locally and pending sync.
  11. Restore network and sync.
  12. Open /admin/submissions and inspect the latest records.

8. Sync Verification

Online report syncs immediately and receives a server receipt.
Offline report is saved locally with pending status.
Pending report retries after connectivity returns.
Web admin shows the answers in table format.
Submission metadata includes officer, device, country tenant, border post, digital address, and GPS fields when permission is granted.
Dashboard aggregates update after synced submissions are accepted.

9. Security And Hosting Checks

Before hosting or sharing a deployed demo, complete these checks:

Change or remove all demo passwords.
Set APP_ENV=production and APP_DEBUG=false.
Use HTTPS and a trusted domain for mobile server URLs.
Run migrations and seeders only against the intended database.
Confirm role scoping for admin users, officer users, countries, border posts, forms, and submissions.
Run the Laravel test suite before handoff.
cd webapp
/opt/homebrew/opt/php@8.4/bin/php artisan test

10. Common Issues

Issue Cause Fix
Phone cannot reach 127.0.0.1 That address points to the phone itself. Use 10.0.2.2 for emulator or the computer LAN IP for a real phone.
Phone cannot reach LAN IP Laravel is bound to localhost. Start Laravel with --host=0.0.0.0 --port=8000.
Login works on web but not mobile Wrong server URL, stale QR password, or token issue. Verify the URL, use the newest QR, or reseed and use Officer123!.
No forms appear on mobile No published form exists for the officer tenant. Run the template seeder or publish a form in /admin/forms.
Submission stays pending Server unreachable, HTTPS/domain issue, or expired token. Check server URL, network, login again, then sync.
Do not expose local demo credentials on a public server. Create named admin and officer accounts for each hosted demo.
